ArbEngine-RS: High-Performance SIMD Processing EngineArbEngine-RS is a low-latency, high-throughput numerical processing engine built in Rust. It leverages SIMD (Single Instruction, Multiple Data) and multi-core parallelism to saturate CPU execution ports, making it ideal for High-Frequency Trading (HFT), signal processing, or large-scale scientific simulations.🚀 Key FeaturesMemory Alignment Control: Custom AlignedBuffer ensures 64-byte alignment, optimizing L1/L2 cache line utilization and preventing "cache line split" penalties.Hardware Acceleration: Utilizes portable_simd (nightly) for architecture-agnostic vectorization (AVX-2, AVX-512, NEON).Massive Parallelism: Powered by Rayon for work-stealing multi-threading.Aggressive Optimization: Implements Manual Loop Unrolling (4x) and branch-minimized masking to maximize Instruction-Level Parallelism (ILP).Zero-Cost Abstractions: Uses trait-based strategies to allow custom logic injection without runtime overhead.🛠 Technical ArchitectureThe engine is designed around a three-tier hierarchy:Storage (AlignedBuffer): Manages raw heap allocation with strict alignment constraints.Logic (SIMDStrategy): A trait defining the vectorized operation (e.g., threshold detection, signal filters).Orchestrator (ArbEngine): Handles the parallel distribution and the unrolled SIMD loop execution.Performance DesignThe engine processes data in chunks. Each chunk is further divided into SIMD vectors. By using as_chunks::<4>(), we reduce the loop bookkeeping overhead and allow the CPU's out-of-order execution engine to process multiple vectors in flight.💻 UsageRust// Example: Detect values exceeding a specific threshold
let detector = VolatilityDetector {
    threshold: Simd::splat(0.5),
};

let engine = ArbEngine::new(detector);
let buffer = AlignedBuffer::<16>::new(1_000_000, 0.1); // 1M elements

engine.run(&buffer, 1024);
println!("Hits: {}", engine.hits.load(Ordering::SeqCst));
